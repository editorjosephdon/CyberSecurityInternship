# Ethical Hacking Task 02: Network Scanning & Service Enumeration

## 🎯 Objective
The purpose of this task is to understand the second phase of Ethical Hacking: Scanning and Enumeration[cite: 4]. This exercise focuses on safely identifying active hosts, open ports, running services, and basic network information using authorized methods on local or permitted systems[cite: 4].

## 📁 Folder Contents

*   **`Nmap_Scan_Report.pdf`** - A comprehensive professional scan report detailing the scan date, target system, commands used, running services, OS detection results, observations, and security recommendations[cite: 4].
*   **`Screenshots/`** - Visual documentation verifying the Nmap installation (`nmap --version`), local machine scans (`nmap localhost`), service version detection (`-sV`), and OS detection (`-O`)[cite: 4].
*   **`Research_Notes.md`** - Detailed analysis of common network ports (such as FTP, SSH, HTTP, HTTPS, RDP) including their protocols, purposes, and common security risks[cite: 4].
*   **`README.md`** - This index file summarizing the task contents.

## 🛠️ Tools Used
*   **Nmap (Network Mapper):** Utilized for network discovery, port scanning, service version detection, and operating system fingerprinting[cite: 4]. 

## 🧠 Key Takeaways
Completing this task provided hands-on experience with discovering a system's attack surface[cite: 4]. By analyzing which ports are open and determining if those services are strictly necessary, I gained a deeper understanding of how misconfigured services can become security risks and the importance of minimizing an environment's exposure[cite: 4].
