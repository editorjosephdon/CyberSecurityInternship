Part B: Password Security Review

1. What makes a strong password?

A strong password is long, complex, and unpredictable. It should contain a mix of uppercase and lowercase letters, numbers, and special characters, avoiding dictionary words or easily guessable personal information.  

2. Why should passwords be unique?

Using unique passwords ensures that if one specific website or service is compromised, attackers cannot use those same credentials to access your other, potentially more sensitive accounts.  

3. What is Multi-Factor Authentication (MFA)?
MFA is a security mechanism that requires users to provide two or more verification factors to gain access to a resource (like an application or online account). This usually combines something you know (a password) with something you have (a code from an authenticator app or a text message).  


4. What are the risks of password reuse?

The primary risk is credential stuffing, where hackers use leaked username and password pairs from one data breach to automatically log into other services.

*Personal Password Policy Enforce a minimum length of 14 characters for all accounts.
*Require a combination of uppercase letters, lowercase letters, numbers, and symbols.  
*Strictly prohibit the use of personal identifiers (names, birthdates, phone numbers).  
*Mandate the use of a reputable password manager to generate and store unique credentials.
*Require MFA to be enabled on any service that supports it, prioritizing authenticator apps over SMS.
